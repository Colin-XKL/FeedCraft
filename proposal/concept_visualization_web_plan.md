# 核心概念可视化 Web 页面规划 (Visualization Web Page Plan)

> 状态：未开始

## 1. 页面定位与目标

**目标**：创建一个独立于文档库的、高度视觉化、交互式的 Web 页面。用最直观的方式向用户（尤其是非技术用户）解释 FeedCraft 架构的运转逻辑。
**受众**：初次接触 FeedCraft，被一堆新名词（AtomCraft, RecipeFeed, TopicFeed）绕晕的用户。
**核心原则**：Less text, more animation.（少字多图，所见即所得）。

## 2. 页面结构与分屏设计

整个页面采用“向下滚动讲述故事（Scrollytelling）”的方式，分为四个主要的区块（Section）。

### Section 1: The Raw Feed (万物起源)

- **视觉隐喻**：水龙头、散落的像素块、或者未经雕琢的矿石。
- **文案**：一切从这里开始。无论是 RSS 链接、一个普通的 HTML 网页，还是搜索引擎的结果，FeedCraft 首先将它们抓取，化作最基础的数据流。
- **动画**：
  - 左侧显示不同形态的图标（网页、JSON、搜索框）。
  - 经过一个扫描仪后，统一变成了标准的、灰色的“数据方块”源源不断地向右流动。
  - 标注：**`RawFeed` (原始源)**。

### Section 2: The Crafts (精雕细琢)

- **视觉隐喻**：流水线上的机械臂、滤网、染色机。
- **文案**：这才是 FeedCraft 的灵魂。像搭积木一样串联你的处理工艺。
- **动画**：
  - 灰色的数据方块流进一条传送带。
  - 经过一个名为 **`AtomCraft (提取正文)`** 的节点，方块变大了（装满了内容）。
  - 经过一个名为 **`AtomCraft (AI 翻译)`** 的节点，方块变色了（完成了翻译）。
  - _交互_：用户可以点击旁边的按钮，随时向传送带上插入新的 Craft 节点，实时看到下方数据方块形态的变化。
  - 标注：多个 AtomCraft 串联，形成了 **`FlowCraft` (组合工艺)**。

### Section 3: The RecipeFeed (打包成型)

- **视觉隐喻**：装箱、贴上专属标签。
- **文案**：将你的源头和加工流水线绑定，就获得了一个专属的、永久可用的新 RSS 源。
- **动画**：
  - 把 Section 1 (水龙头) 和 Section 2 (传送带) 框在一个虚线框里。
  - 在这个框的出口，生成了一个闪亮的 RSS 订阅图标。
  - 标注：**`RawFeed` + `FlowCraft` = `RecipeFeed` (配方源)**。

### Section 4: The TopicFeed (百川汇海)

- **视觉隐喻**：多条河流汇聚成一个湖泊、或者多条流水线汇入一个巨大的漏斗。
- **文案**：信息太多？把它们合并起来。自动去重，保留精华，构建你的专属主题资讯站。
- **动画**：
  - 画面上同时出现 3 个正在运行的 `RecipeFeed`（产出着不同颜色的数据块）。
  - 它们全部流向一个巨大的容器（Aggregator 聚合器）。
  - 容器里出现闪烁的动画（模拟对比和去重，一些重复的方块被剔除）。
  - 最终在容器底部，流出一条更加宽广、色彩丰富的河流。
  - 标注：**多个 `RecipeFeed` 合并 = `TopicFeed` (主题源)**。

## 3. 技术实现选型建议

为了实现上述这种流畅的数据流动动画和“乐高式”的拼装感，建议在现有的前端技术栈上采用以下方案：

1.  **框架**：使用当前项目可能已有的 React 或 Vue 作为基础。
2.  **动画核心库**：
    - **Framer Motion** (React) 或 **VueUse Motion**：用于处理页面滚动时的渐入渐出和组件平滑过渡。
    - **React Flow / Vue Flow**：**强烈推荐**。因为未来的可视化编排节点编辑器本来就需要用到它。我们可以直接用这个库，加上带动画的 Edge (连线)，完美模拟数据在节点间流动的“流水线”效果。
3.  **CSS 样式**：TailwindCSS（方便快速构建精致的卡片和布局）。

## 4. 落地步骤

1.  **线框图设计 (Wireframe)**：在 Figma 或 Excalidraw 中画出这四个 Section 的静态草图。
2.  **动效原型设计**：确定“数据方块 (CraftArticle)”如何在屏幕上移动（是用 CSS Keyframes 还是 React Flow 的 Animated Edges）。
3.  **开发与内嵌**：
    - 可以作为一个单独的静态路由放在官网 (`doc-site/src/pages/concepts.astro` 中创建一个定制页面)。
    - 或者直接做在应用的登录后 Welcome 首页。
